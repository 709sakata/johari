# じょはり (Johari)

まだ知らない自分に出会う思考の窓。あなたのアイデンティティをプロデュースし、デジタルツインとして磨き上げるためのプラットフォームです。

## コンセプト：アイデンティティのデジタルツイン

「じょはり」は、心理学モデル「ジョハリの窓」をベースに、超デジタル時代における**「アナログ（現実）とデジタルのアイデンティティの乖離」**を埋めることを目指しています。

1.  **開放の窓 (Open Area)**: 自分も他人も知っている自分。
2.  **盲点の窓 (Blind Area)**: 自分は気づいていないが、他人は知っている自分。
3.  **秘密の窓 (Hidden Area)**: 自分は知っているが、他人は知らない自分。
4.  **未知の窓 (Unknown Area)**: 自分も他人もまだ気づいていない、可能性としての自分。

### 「じょはり」が解決すること：ユーザーをプロデュースする

現代において、デジタル上の公開情報から「あなたがどう見えているか」は、経済的にも人格的にも極めて重要な「事実」です。

「じょはり」は単なるメモ帳やSNSではなく、**「あなたをプロデュースするエージェント」**として機能します。

- **事実への問い**: 「デジタルの公開情報（スレッドの内容や登録された外部リンク）からは、あなたはこう見えていますが、それは "事実" ですか？」という問いを立てる。
- **メタ認知の深化**: 問いに答えるプロセスを通じて、ジョハリの窓（特に盲点と秘密の窓）を意識し、自己理解を深める。
- **正しい伝達**: 第三者があなたについて調べた際、あなたのアイデンティティが「正しく」伝わる状態を構築する。

## じょはりの目的

- **アイデンティティの外部化**: 思考の断片をスレッド（Scrap）として蓄積し、さらに登録された外部リンク（GitHub, Twitter, ブログ等）の公開情報と統合することで、デジタルツインの素材とする。
- **発問によるプロデュース**: AIや他者からの「問い」を通じて、自分一人では到達できなかったメタ認知を得る。
- **未知の自分（Unknown Area）の発見**: 蓄積された思考を編纂し、まだ見ぬ自分の可能性を社会に正しく提示する。

## 主な機能

- **スレッド（Scrap）作成**: 思考の断片をタイトルと内容で記録。
- **リアルタイム対話**: Firebase Firestore を利用したリアルタイムなコメント・返信機能。
- **Markdown サポート**: 数式（KaTeX）、コードハイライト（Highlight.js）、GitHub Flavored Markdown に対応。
- **ステータス管理**: スレッドを「オープン（進行中）」または「クローズ（完了）」として管理。
- **プロフィール機能**: 自己紹介（50文字制限の「続きを読む」機能付き）やアイコンの設定。
- **マイページ**: 自分が作成したスレッドの管理と、外部サイトへの埋め込み用 iframe コードの生成。
- **公開プロフィール**: 他のユーザーの思考一覧を閲覧可能。
- **レスポンシブデザイン**: モバイル・デスクトップ両対応のモダンな UI。

## 技術スタック

- **Frontend**: React 19, Vite, Tailwind CSS 4
- **Backend**: Express (Vite Middleware)
- **Database/Auth**: Firebase (Firestore, Authentication)
- **Animation**: Motion (motion/react)
- **Icons**: Lucide React
- **Markdown**: react-markdown, remark-gfm, rehype-highlight, rehype-katex

## セットアップ

### 環境変数

以下の環境変数を設定する必要があります。

- `GEMINI_API_KEY`: Gemini API を利用する場合に必要（現在は主にプラットフォーム側で管理）。
- `GOOGLE_MAPS_PLATFORM_KEY`: 地図機能を利用する場合に必要。
- Firebase 設定 (`firebase-applet-config.json`):
  - `apiKey`
  - `authDomain`
  - `projectId`
  - `appId`
  - `firestoreDatabaseId`

### 開発コマンド

```bash
# 依存関係のインストール
npm install

# 開発サーバーの起動
npm run dev

# ビルド
npm run build

# 型チェック（Lint）
npm run lint
```
